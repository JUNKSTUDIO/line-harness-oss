# スタンプカード LIFF画面 UI/UX アドバイス

対象: ユーザー用 LIFF画面 (`?page=stamp-card`)。表示する3要素 — ① 現在のランク、② ポイント数、③ 期限が近い場合の延長ボタン — について、レイアウトと文言の指針をまとめる。実装の骨格は `apps/worker/src/client/stamp-card/`（本ドキュメントと同時に追加）。

## 全体方針

- 既存の `event-booking` / `salon-booking` LIFF画面と同じ「LINE緑 (#06C755) + 角丸カード + 上部固定ヘッダー」の見た目に統一する。スタンプカードだけ別デザイン言語にすると、同じLINEアカウント内で世界観が割れる。
- 1画面1メッセージ。情報を詰め込まず、「今ココ」「あと何個」「いつまで」の3つだけを大きく見せる。
- 数字（スタンプ数・残り日数）は本文中で最も大きいフォントサイズにする。ユーザーが一番見たいのはここ。

## ① 現在のランク表示

**レイアウト**
- 画面最上部、ヘッダー直下にランクバッジを大きく配置（カードのhero的役割）。
- ランクごとに背景グラデーションを変える（要件どおり）。例:
  - ブロンズ: `#fef3c7 → #fde68a`（くすんだ金茶）
  - シルバー: `#f1f5f9 → #cbd5e1`（グレー系）
  - ゴールド: `#fef9c3 → #facc15`（明るい金）
  - プラチナ: `#e0e7ff → #a5b4fc`（淡い紫がかった銀）
- ランクバッジ内に「現在のランク」ラベル（xs, グレー）+ ランク名（lg, bold, 黒）を縦並びで。
- ランク機能がOFFの店舗（`rank_enabled = 0`）ではこのセクション自体を非表示にし、スタンプ進捗だけを見せる。ランク無し運用なのにランク欄が出るとユーザーが混乱する。

**文言**
- ラベルは「現在のランク」固定。「あなたのランク」等に揺らがせない（再訪時に同じ言葉で認識させる）。
- 次のランクがある場合、バッジの下に小さく「次のランク: シルバー」を添える。最終ランクの場合は「最高ランクです🎉」で締め、ゴールがないモチベーション切れを防ぐ。

## ② ポイント数（スタンプ進捗）表示

**レイアウト**
- ランクバッジの下に、円形または横長バーのプログレスメーターを1つ。要件の「マトリクス状/メーター状」のうち、来店ごとに数字が動く店はマトリクス（スタンプ絵が並ぶ）の方が「貯まってる感」が出やすく、金額ごと付与の店はバー（連続値）の方が自然。`stamp_rule_type` に応じてどちらを出すかをコンポーネント側で自動選択する。
- メーター中央 or 右上に `現在値 / ゴール` を大きい数字で（例: `7 / 10`）。
- メーターの下に1行、「あと◯個でクリア（次のランクへ）」。ゴール到達済みなのに次ランクがない（最終ランク＆flatゴール）場合は「クーポンと交換できます」に文言を切り替える。

**文言**
- 「あと3個でクリア（次のランクへ）」— 要件の表現をそのまま採用。"クリア" は到達後の演出（紙吹雪アニメ等、実装余地）と語感を合わせやすいので維持する。
- 0個の状態（発行直後）でも「あと10個でクリア」が出ることを確認する。「0/10」だけだと初見ユーザーがゴールの意味を理解しにくい。

## ③ 延長ボタン（期限が近い場合のみ）

**表示条件**
- `canExtend`（API側で `expiration_extended=0` かつ 残り日数 ≤ `reminder_days_before`）が true のときだけ描画する。それ以外では完全に非表示（disabledボタンとして見せない）。中途半端に「期限はまだ大丈夫です」のような薄い表示にすると、本来目立たせたい来店動線への集中を削ぐ。
- 期限が無い設定（`card_expiry_months` 未設定）のショップでは、このセクション自体を考慮不要（API が `expiresAt: null` を返す）。

**レイアウト**
- スタンプ進捗セクションの下、画面下部固定ではなく本文内に置く（誤タップ防止。固定フッターに置くと「予約する」ボタン等と並んで誤操作されやすい）。
- 黄〜オレンジ系の警告トーン（背景 `#fff7ed`、文字 `#c2410c`）の薄いカードで「期限間近」を視覚的に区別する。プッシュ通知のFlexメッセージ（`card-coupon-notifier.ts` のヘッダー配色）と完全に統一し、「あ、さっき来た通知のあれだ」と即座に紐づけさせる。
- ボタンは secondary スタイル（予約ボタンより視覚的に一段下げる）。延長は「本来は来店してほしいが、それが難しい人向けの救済」なので、予約導線より目立たせすぎない。

**文言（最重要・要件どおりの文言を踏襲）**
- ボタン本体: 「どうしても来店できない方はこちら（1回限定で1週間延長）」
  - 「1回限定」を必ずボタン文言に出す。タップ前に再利用不可だと伝えることが、タップ後の「もう使えません」エラーに対する納得感を生む。
- タップ後（成功）: 「有効期限を1週間変更しました。ご来店を心よりお待ちしております！」
- タップ後（既に延長済み — 通常は表示条件で防がれるが、複数タブ/通知の押し直し等で到達するケース）: 「このカード/クーポンは既に一度延長されています」
- 延長成功後は即座にボタンを非表示にし、代わりに「延長済み（次の期限: 2026年7月8日）」という確定情報に置き換える。同じ画面でもう一度押せそうに見える状態を残さない。

## 補足: クーポン一覧へのリンク

- スタンプ進捗カードの下に、テキストリンク1行「保有しているクーポンを見る →」を置く（`/api/liff/coupons/me` を呼ぶ別画面に遷移）。ボタンとして強く出すと上記①②③の主役を奪うため、リンクとして控えめに配置する。
